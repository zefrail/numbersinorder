# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** `Activity`
*   **Назначение:** Сердце игры. Здесь генерируются числа, обрабатываются нажатия и проверяется правильность порядка.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение |
| :--- | :--- | :--- |
| `numberButtons` | `Button[]` | Массив из 5 кнопок с числами. |
| `starViews` | `ImageView[]` | Звездочки прогресса. |
| `numbers` | `ArrayList` | Список случайных чисел, отсортированных по порядку. |
| `currentIndex` | `int` | Индекс числа, которое мы ищем сейчас (0, 1, 2...). |

## 3. Методы класса

### Имя метода: `generateNumbers`
*   **Что делает:** Создает 5 уникальных случайных чисел от 10 до 99 и сортирует их. Это "эталон", с которым мы будем сравнивать нажатия игрока.

### Имя метода: `handleButtonClick`
*   **Что делает:** Сравнивает число на кнопке с тем, которое должно быть следующим по порядку. Если верно — вызывает успех, если нет — ошибку.

## 4. Жизненный цикл
*   **`onCreate()`**: Начало игры, загрузка звуков.
*   **`onDestroy()`**: Очистка `SoundPool` (освобождение памяти от звуков).

## 5. Взаимодействие с интерфейсом (UI)
*   **Массивы View:** Использование массивов (`Button[]`) позволяет управлять пятью кнопками в цикле, не дублируя код 5 раз.
*   **Анимации:** Тряска кнопки при ошибке и пульсация как подсказка.

## 6. Взаимодействие с другими компонентами
*   **`AlertDialog`**: Всплывающие окна с кнопками "Начать заново" или "Настройки".

## 7. Общая логика
1. Генерируем числа.
2. Перемешиваем их на кнопках.
3. Игрок должен нажать самое маленькое, затем чуть больше и т.д.
4. Правильное нажатие — загорается звезда. Ошибка — минус жизнь.

## 8. Объяснение простыми словами
**Аналогия:** Перед вами 5 закрытых коробок. Вам нужно открыть их по очереди: от самой легкой до самой тяжелой. Если ошиблись — звучит сирена!

**Совет:** В коде используется `Collections.shuffle()`. Это отличный способ сделать игру непредсказуемой: числа те же, но их места на экране всегда разные.
