# Описание класса: Start

## 1. Общая информация
*   **Имя класса:** `Start`
*   **Тип:** `Activity` (Главное окно приложения)
*   **Назначение:** Это "Splash Screen" (экран приветствия). Его задача — поприветствовать пользователя, показать красивую анимацию в течение нескольких секунд и затем автоматически перенаправить его на экран настроек.
*   **Взаимодействие:** Запускается при открытии приложения. Через определенное время использует `Intent` (объект-намерение для связи между экранами), чтобы открыть `SettingsActivity`.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `titleTextView` | `TextView` | Отображает название приложения. | `onCreate()` |
| `animatedTextView` | `TextView` | Текстовое поле, которое растет в ширину и меняет цвет. | `onCreate()` |
| `handler` | `Handler` | Инструмент для планирования задач на будущее. | `onCreate()`, `onDestroy()` |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Хранит данные для восстановления состояния экрана. |
*   **Что делает:** 
    1.  Устанавливает внешний вид из XML-файла.
    2.  Находит элементы интерфейса (`TextView`).
    3.  Настраивает анимацию появления (прозрачности) для заголовка.
    4.  Вычисляет ширину экрана и создает аниматор ширины (`widthAnimator`) для прогресс-бара.
    5.  Создает аниматор цвета (`colorAnimator`) для плавного перехода от желтого к зеленому.
    6.  Запускает все анимации.
    7.  Использует `handler.postDelayed`, чтобы подождать 7 секунд и перейти в `SettingsActivity`.
*   **Когда вызывается:** Автоматически системой Android при создании экрана.

### Имя метода: `onDestroy`
*   **Тип:** `protected`
*   **Что делает:** Удаляет все запланированные задачи из `handler`.
*   **Когда вызывается:** Перед закрытием экрана.
*   **Важно понимать:** Это нужно для предотвращения "утечек памяти". Если не очистить handler, приложение может попытаться выполнить задачу, когда экран уже закрыт, что вызовет ошибку.

## 4. Жизненный цикл
*   **`onCreate()`**: Инициализация и запуск анимаций.
*   **`onDestroy()`**: Очистка ресурсов (handler).

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `TextView`.
*   **Связь:** `findViewById` связывает дизайн из XML с кодом на Java.
*   **Анимации:** Используется `ObjectAnimator` для динамического изменения свойств (ширина, цвет, прозрачность).

## 6. Взаимодействие с другими компонентами
*   **Переходы:** `Intent` для перехода к `SettingsActivity`.
*   **`finish()`**: Закрывает текущий экран, чтобы пользователь не мог вернуться к заставке кнопкой "Назад".

## 7. Общая логика
При запуске экран плавно показывает название и "заполняет" полоску цветом. Через 7 секунд заставка закрывается и открывается меню настроек.

## 8. Объяснение простыми словами
**Аналогия:** Представьте заставку как трейлер перед фильмом. Вы сидите в кино, свет гаснет, и вы видите логотип студии. Вам не нужно ничего нажимать — как только ролик закончится, начнется сам фильм (или появится меню).

**Совет:** В коде используется `Handler` для отсчета времени. Это классический способ сделать паузу в Android.
